# 📜 创作者守则 | Creator Code of Conduct

> **Origin of Miracles** 社区创作规范

---

## 🎯 核心原则

本项目致力于为所有年龄段的玩家提供**健康、积极、有趣**的游戏体验。所有贡献内容必须遵守以下原则：

### ✅ 我们欢迎

- 忠于原作人设的角色配置
- 高质量的模型和动画作品
- 富有创意的对话设计
- 尊重他人的讨论和反馈
- 对新贡献者的友善指导

### ❌ 我们禁止

以下内容**严格禁止**出现在本项目中：

#### 1. 成人内容 (R18/18G)
- ❌ 任何性暗示、色情或擦边球内容
- ❌ 裸露、不雅姿势或服装
- ❌ 成人向对话或剧情

#### 2. 暴力血腥
- ❌ 过度血腥、残暴的视觉呈现
- ❌ 恐怖、惊悚的模型设计
- ❌ 暴力倾向的对话内容

#### 3. 政治敏感
- ❌ 任何政治立场、政治人物
- ❌ 意识形态宣传
- ❌ 地区争议性内容

#### 4. 其他不当内容
- ❌ 种族歧视、性别歧视
- ❌ 仇恨言论、人身攻击
- ❌ 侵犯他人版权的作品
- ❌ 恶意代码或病毒

---

## 📝 内容审核标准

### 人格配置 (Persona Configs)

**✅ 合格示例：**
```json
{
  "system_prompt": "你是阿罗娜，温柔体贴的 AI 助手，总是关心老师...",
  "example_dialogues": [
    {
      "user": "你好",
      "assistant": "老师好~！今天也要加油哦！"
    }
  ]
}
```

**❌ 不合格示例：**
- 包含暴力、色情暗示的对话
- 涉及政治、宗教的讨论
- 使用侮辱性、攻击性语言

### 模型与纹理

**✅ 合格标准：**
- 忠于原作角色设计
- 服装得体，无暴露或不当姿势
- 纹理清晰，无不良图案

**❌ 不合格特征：**
- 过度裸露的服装设计
- 血腥、恐怖的视觉元素
- 政治符号、敏感标识

---

## 🔍 审核流程

所有提交的 Pull Request 将经过以下审核：

1. **自动检测**：脚本扫描敏感关键词
2. **人工审核**：维护团队逐一检查内容
3. **社区反馈**：接受社区成员的举报

如果内容违反守则，PR 将被**拒绝**并要求修改。

---

## 🚨 违规处理

### 首次违规
- 警告并要求修改内容
- 提供改进建议

### 重复违规
- 禁止参与贡献
- 封禁账号（严重情况）

### 严重违规（R18/政治/恶意代码）
- **立即封禁**
- 移除所有相关内容
- 向 GitHub 平台举报

---

## 💬 举报不当内容

如果发现违规内容，请：

1. **提交 Issue**：[Issues 页面](https://github.com/Origin-of-Miracles/Anima-Assets/issues)
2. **联系维护者**：通过 Email 或 Discord 私信
3. **提供证据**：附上截图或文件链接

我们将在 **24 小时内**响应并处理。

---

## 🌟 良好创作建议

### 人格配置
- 保持角色一致性，参考官方设定
- 对话自然流畅，符合角色性格
- 避免过度戏剧化或 OOC（脱离人设）

### 模型制作
- 使用高质量纹理，注意细节
- 动画流畅自然，避免卡顿
- 测试模型在游戏中的表现

### 社区互动
- 友善对待新手，耐心解答问题
- 尊重他人创作，给予建设性反馈
- 分享经验，帮助社区共同进步
